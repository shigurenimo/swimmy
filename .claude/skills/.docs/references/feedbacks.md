# feedbacks/

顧客の声をテーマ別に管理する。voiceスキルで追記する。

## slug.md

1テーマ1ファイル。関連する声をまとめる。

FrontMatter(任意):

```yaml
---
notion-page-id: {Notion Page ID}
---
```

本文:

```md
# {課題テーマ名}

## 声
- {原文の要点} [^1]
- {原文の要点} [^2]

## 課題
- {分析した課題1}
- {分析した課題2}

[^1]: 「{原文}」({誰}, {YYYY-MM-DD}, {ソース})
[^2]: 「{原文}」({誰}, {YYYY-MM-DD}, {ソース})
```

脚注に原文・日付・ソースをまとめる。本文は要点だけ。

## index.md

slug.mdから再生成するキャッシュ。手で編集しない。件数の多い順。

```md
# フィードバック

## {課題テーマ名} ({N}件)
{課題の要約1文}

## {課題テーマ名} ({N}件)
{課題の要約1文}。ビジョン外
```

## ルール

- 声の原文を残す。要約しない
- 新しい声は既存テーマに追記。類似テーマがなければ新規slug.md作成
- ビジョン外の要望には `## 課題` にその旨を明記
- テーマの課題がproduct.mdの「解決する問題」に昇格したら `## 課題` に明記
