# 新機能のアイデア創出

新規機能のアイデアを創出し、仕様計画を作成する。

## 起動方法

引数なし。製品を理解してアイデアを創出する。

## プロセス

### 仕様書の構造理解

`docs` スキルで仕様書の構造とルールを理解する。

### 既存製品の理解とアイデア創出

`docs` スキルで製品を理解する。

**新機能のアイデア創出**

製品の成長に貢献する機能改善を考える:

- ユーザー体験の向上
- パフォーマンスの改善
- セキュリティの強化
- 保守性の向上
- 新しい価値提供

### 仕様レベルの詳細計画作成

`superpowers:brainstorming` スキルを使用して、以下を明確にする:

- **何を作るのか**: 機能の概要
- **何のために作るのか**: ビジネス上の目的、ユーザーへの価値
- **誰が使うのか**: 対象ユーザー
- **どうなれば成功か**: 受け入れ条件

### 提案の報告

以下の形式で報告する:

```
新機能を提案します。

## タイトル

{タスク名}

## 概要

{機能の概要}

## 目的

{ビジネス上の目的、ユーザーへの価値}

## 対象ユーザー

{誰が使うのか}

## 受け入れ条件

- {条件1}
- {条件2}
- {条件3}

## 推奨優先度

{高/中/低}

## ビジネス価値

{価値の説明}
```

技術的な内容（ファイル名、関数名、コードなど）は含めない。ユーザー目線の言葉で記載する。

仕様計画を成果物として提出して終了する。実装は行わない。
