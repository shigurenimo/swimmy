# ドメイン関連

概念・ルール・用語のファイル群。

## actors.md

固定ロール（ユースケースの主語）の定義。

書くこと:

- 冒頭に設計背景（なぜこのロール構成か）
- 各アクターに「何ができるか」「なぜこのロールがあるか」

動的ロール（管理画面で作成・編集できるロール）は models/ に Role として書く。

## relationships.md

概念の関係をテーマごとに図示。

```markdown
## 全体

（全体の関係図）

## 投稿

（投稿に関する関係図と説明）

## エンゲージメント

（エンゲージメントに関する関係図と説明）
```

全体図を先頭に、詳細をテーマで分割。

## models/{model}.md

集約ルート（Aggregate Root）ごとの構造と不変条件。1モデル1ファイル。

書くこと:

- 集約ルートとその境界内のエンティティを図示
- 不変条件（ビジネスルール）
- 状態遷移がある場合はライフサイクル
- 「なぜこの境界か」「なぜこの制約か」

```markdown
# Post

Post（ルート）
  ├── Reaction
  └── Comment

Reaction と Comment は Post なしに存在できない。

## 不変条件

**タイプは変更不可**

後から変更できると...
```
