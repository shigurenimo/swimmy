# ファイルの役割

product/ 内のファイル構成。基本的にはすべて作成する。

## ファイル

| ファイル | 何を記述するか | 省略条件 |
|---------|---------------|---------|
| overview.md | 製品の目的・背景 | - |
| sitemap.md | 全ルートの一覧 | - |
| actors.md | アクター（ロール） | - |
| relationships.md | 概念の関係 | - |
| architecture.md | 技術選定・構成 | CLAUDE.md に記載済みの場合 |
| operations.md | 運用（監視、デプロイ、アラート） | 運用情報がない場合 |
| database.md | スキーマ設計の意図 | DB を使用しない場合 |
| personas.md | ペルソナ | ペルソナ設計がない場合 |
| api.md | API設計の意図 | API がない場合 |

## ディレクトリ

| ディレクトリ | 何を記述するか | 省略条件 |
|-------------|---------------|---------|
| pages/ | 画面ごとの設計意図 | - |
| features/ | 機能の概要・目的 | - |
| stories/ | ユーザーストーリー | ストーリー設計がない場合 |
| models/ | 集約ルートの構造・不変条件 | モデル設計がない場合 |
| services/ | ユースケース | サービス層がない場合 |
| policies/ | 横断的な制約やルール | ポリシーがない場合 |
| integrations/ | 外部サービス連携 | 外部連携がない場合 |
| issues/ | 仕様の問題点 | - |
