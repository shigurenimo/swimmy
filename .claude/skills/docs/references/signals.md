# signals/

顧客と社内の声をテーマ別に管理する。voiceスキルで追記する。

## slug.md

1テーマ1ファイル。関連する声をまとめる。

FrontMatter(任意):

```yaml
---
notion-page-id: { Notion Page ID }
---
```

本文:

```md
# {課題テーマ名}

最終更新: YYYY-MM-DD

ここから --- までは人間が自由に書く。Claude は読み取りのみで書き換えない。
このテーマに関する独自メモ、判断、ステータス、社内コメントなど。

---

ここから下は triage スキルが再生成する。手で書いた変更は次回上書きされる。

{声の要点を文章で記述} [^1]。{別の声があれば続けて記述} [^2]。

## 課題

- {分析した課題1}
- {分析した課題2}

[^1]: 「{原文}」({誰}, {YYYY-MM-DD}, {ソース})

[^2]: 「{原文}」({誰}, {YYYY-MM-DD}, {ソース})
```

声は箇条書きではなく文章で記述する。脚注に原文・日付・ソースをまとめる。

## ゾーン分離 (`---`)

frontmatter とは別に、`# {テーマ名}` の直下に「人間ゾーン」、その後の `---` で「Claude ゾーン」に分ける。Claude (triage スキル) は `---` から下のみ書き換える。

frontmatter があるファイルでの `---` の出現順:

1. frontmatter 開始
2. frontmatter 終端
3. 人間ゾーンと Claude ゾーンの分離 (これが上書き対象の境界)

frontmatter が無いファイルでは、本文先頭から最初の `---` が分離線。

人間ゾーンが空でもよい (空行のみで `---` だけ置く)。

## index.md

slug.md から再生成するキャッシュ。手で編集しない。カテゴリ別ナラティブと脚注で構成する。

### 再生成手順

指定した signals ディレクトリ配下の slug.md を全て読み込む（index.md は除く）。

各 slug.md からテーマ名、声の要約、課題、声の件数（脚注数）を抽出する。

声と課題の内容からカテゴリを導出する。カテゴリは事前に固定せず、その product で自然に立ち上がる分類を使う。

カテゴリごとにナラティブ本文を書き起こす。同一カテゴリの複数テーマを文章でつなげる。

冒頭にサマリ段落を置く。全体傾向、注目テーマ、ビジョンとの整合を記述する。

末尾に脚注を並べる。形式は `[^N]: {slug}.md ― {テーマ要約}（{N}件）`。

`signals/index.md` を上書き保存する。

### フォーマット

```md
# フィードバック

{冒頭サマリ段落。全体傾向・注目テーマ・ビジョンとの整合}

## {カテゴリ名}

{そのカテゴリに属する複数テーマをナラティブで記述} [^1]。{別テーマの記述} [^2]。

## {カテゴリ名}

{本文} [^3]。

---

[^1]: {slug}.md ― {テーマ要約}（{N}件）

[^2]: {slug}.md ― {テーマ要約}（{N}件）
```

### ルール

カテゴリ数は5から10程度を目安にする。

slug.md は編集しない。index.md のみ上書きする。

脚注番号は本文の出現順で付ける。

ビジョン外のテーマは本文でその旨を明示する。

声数の多いテーマを優先して記述の粗密を調整する。

既存 index.md の構成は参照せず、slug.md から新規生成する。

## ルール

- 声の原文を残す。要約しない
- 新しい声は既存テーマに追記。類似テーマがなければ新規slug.md作成
- ビジョン外の要望には `## 課題` にその旨を明記
- テーマの課題がindex.mdの「解決する問題」に昇格したら `## 課題` に明記
- Claude ゾーンのみ書き換える。人間ゾーン (`---` より上) には触らない
- `最終更新` 行は人間ゾーンの先頭に置く。Claude ゾーンを再生成したときに今日の日付に更新してよい
