# index.md

製品の全コンテキストを1ファイルに集約する。小さく保つ(100行以下目安)。

各セクションは箇条書き。長文を書かない。

```md
# {製品名}

## ジョブ(顧客が片付けたいこと)
- {ユーザー種別A}: {片付けたいこと}
- {ユーザー種別B}: {片付けたいこと}

## ビジョン
{一文で書く}

## 解決する問題
- {問題1}
- {問題2}

## ターゲット
- {ユーザー種別A}: {特徴}
- {ユーザー種別B}: {特徴}

## 原則
- {判断に迷ったときの基準1}
- {判断に迷ったときの基準2}

## やらないこと
- {境界1}
- {境界2}

## 技術スタック
- {技術}: 理由: {選定理由}

## 重要な判断履歴
- {過去の大きな判断を簡潔に}
```

## ルール

- ジョブ、ビジョン、原則、やらないこと → 人間の意思。Claudeが勝手に変えない
- 解決する問題 → フィードバックから分析して更新してよい
- 技術スタック → 選定理由を必ず1文つける
- セクションを追加しない。この構造を守る
