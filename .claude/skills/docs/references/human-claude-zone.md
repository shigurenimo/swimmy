# 人間ゾーンと Claude ゾーン

`signals/{slug}.md` `backlogs/{slug}.md` は、人間が自由に書く領域と Claude が再生成する領域を `---` で分ける。triage スキルは Claude ゾーンのみ書き換え、人間ゾーンには触らない。

## ゾーン分離（`---`）

`# {テーマ名}` の直下に「人間ゾーン」、その後の `---` で「Claude ゾーン」に分ける。

frontmatter があるファイルでの `---` の出現順。

- frontmatter 開始
- frontmatter 終端
- 人間ゾーンと Claude ゾーンの分離（これが上書き対象の境界）

frontmatter が無いファイルでは、本文先頭から最初の `---` が分離線。

人間ゾーンが空でもよい（空行のみで `---` だけ置く）。

## ルール

- Claude（triage スキル）は `---` から下のみ書き換える
- 人間ゾーンには Claude は触らない。経営判断、優先度メモ、社内コメント、ステータス変更はここに置く
- `最終更新` 行は人間ゾーンの先頭に置く。Claude ゾーンを再生成したときに今日の日付に更新してよい
- 人間ゾーンを誤って上書きしないよう、再生成前に `---` の位置を確認する

## ファイル別の中身

各ファイルの本文構造とフィールドは [signals.md](signals.md) と [backlogs.md](backlogs.md) を参照。
