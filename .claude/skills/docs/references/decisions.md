# decisions/ (ADR)

1判断1ファイル。番号付き。

ファイル名: `NNN-slug.md`(例: `001-use-oauth.md`)

```md
# NNN: {タイトル}

## 状況

{なぜこの判断が必要になったか}

## 判断

{何をどうするか}

## 理由

- {選択肢A}: {却下した理由}
- {選択肢B}: {却下した理由}
- {採用した選択肢}: {採用した理由}

## 結果

- {何が変わったか}
- {トレードオフ}
```

## ルール

- 番号は連番。欠番は作らない
- 各セクション(状況、判断、理由、結果)を必ず書く
- 理由には却下した選択肢も書く
- supersede(上書き)する場合: `## 状況`に「ADR-NNNを置き換える。理由:」と明記
- 以下のとき新しいADRを作る:
  - 技術選定(ライブラリ、フレームワーク、サービス)
  - 機能の追加・廃止
  - アーキテクチャの変更
  - ビジネスルールの変更
- 以下のときはADRを作らない:
  - バグ修正
  - リファクタリング(設計判断を変えない場合)
  - 軽微なUI変更
