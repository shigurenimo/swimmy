# sources/

決定の根拠になる一次情報を置く。議事録・要件書・客先配布資料・外部仕様書など、後から「なぜそう決めたか」を辿るための痕跡。

`decisions/`（ADR）や `milestones.md`、`signals/` から `[[sources/...]]` で逆リンクすることで、判断の出所がグラフ上に残る。

## ディレクトリ構成

```
sources/
  minutes/                  議事録
    YYYY-MM-DD-{topic}.md
  materials/                配布物（PDF・スクショ・要件書のコピー）
  {要件書名}.md
  {外部仕様書名}.md
```

## 議事録（minutes/）

ファイル名は `YYYY-MM-DD-{topic}.md`。複数の議事録を時系列で辿れるよう日付プレフィックスを必須にする。`{topic}` は短い英字スラッグまたは関係者名（例: `2026-05-08-miyagi`、`2026-05-28`）。

本文は最小フォーマット。

```md
# {YYYY-MM-DD} {打合せ名}

参加: {参加者}
場所: {対面 / オンライン}

## 決まったこと

- {決定事項。decisions/ や milestones に転記される}
- {決定事項}

## 議論

- {検討の経過。意見の対立、見送られた案}

## 持ち帰り

- {誰}: {次までにやること}
```

## ルール

- 議事録は短く要点だけ。発言の全文起こしは別ツールで。意思決定の素材としての痕跡が残ればよい
- 「決まったこと」は `decisions/NNN-{slug}.md` に正式な ADR として再構成し、議事録から `[[decisions/NNN-{slug}]]` を張る
- 議事録から `milestones.md` の該当日付セクションへ wikiリンクし、逆向きにも `[[sources/minutes/{file}]]` で張る
- 客先からもらった要件書・PDF・スクショは `sources/materials/` に置き、本文 md からリンクする
- 同じ topic を複数回打合せた場合はファイルを分け、`2026-05-28-followup` のようにする。1ファイルに継ぎ足さない

## 何を sources/ に置くか

置く。

- 議事録（社内・社外）
- 顧客から提供された要件書、業務手順書、業界仕様書のコピー
- 営業資料の配布版

置かない。

- 個人のメモや調査ログ → `notes/`
- 顧客の口頭発言の整理 → `signals/`
- 正式に決定された意思 → `decisions/`

sources/ は「外から入ってきた事実の保管庫」。Claude は内容を要約・整理しない。ファイル名と冒頭の概要だけで検索できるようにする。
